- Simplicity over cleverness.

# Sprint Rules

Every sprint must include:

- Working code
- File paths
- Explanation
- Verification steps

Nothing is considered complete until:

- Builds successfully
- Typecheck passes
- Lint passes
- Tests pass (when applicable)
- Reviewed
- Committed

# Coding Rules

- Never use `any`.
- Never call `Date.now()` inside the domain layer.
- Never bypass the repository layer.
- Never modify unrelated files.
- Keep functions small.
- Prefer pure functions.
- Explain architectural decisions.
